# 启动脚本说明

本项目提供了三种启动方式，用于同时启动前后端服务。

## 方式一：使用 start.bat（推荐 - 查看日志）

**优点：**
- ✅ 双击即可运行，无需额外环境
- ✅ 可以实时查看服务运行日志
- ✅ 便于调试和排查问题

**使用方法：**
1. 双击 `start.bat` 文件
2. 会自动打开两个命令行窗口分别运行前后端
3. 可以在窗口中实时查看日志输出

**适用场景：**
- 开发调试阶段
- 需要实时查看日志
- 排查问题

---

## 方式二：使用 start_silent.bat（推荐 - 静默运行）

**优点：**
- ✅ 不弹出任何窗口，干净整洁
- ✅ 后台运行，不影响其他工作
- ✅ 日志保存到文件，可随时查看

**使用方法：**
1. 双击 `start_silent.bat` 文件
2. 服务将在后台静默运行（无窗口）
3. 日志文件保存在 `logs/` 目录：
   - `logs/backend.log` - 后端日志
   - `logs/frontend.log` - 前端日志

**适用场景：**
- 日常使用，不需要看日志
- 希望保持桌面整洁
- 长期运行服务

---

## 方式三：使用 start.py（跨平台）

**优点：**
- ✅ 跨平台支持（Windows/Linux/Mac）
- ✅ 更强大的错误处理
- ✅ 易于扩展和维护

**使用方法：**
```bash
python start.py
```

**适用场景：**
- 需要跨平台运行
- Linux/Mac 开发环境
- 需要更复杂的启动逻辑

---

## 启动后的服务地址

- **后端 API 文档**：http://127.0.0.1:8000/docs
- **前端应用**：http://localhost:5173

## 停止服务

### 如果使用 start.bat：
关闭对应的命令行窗口即可停止对应的服务。

### 如果使用 start_silent.bat：
1. 双击运行 `stop.bat`（推荐）
2. 或者使用任务管理器结束 Python 和 Node.js 进程
3. 或者使用以下命令：
   ```bash
   # 停止占用 8000 端口的进程（后端）
   netstat -ano | findstr :8000
   taskkill /F /PID <进程ID>
   
   # 停止占用 5173 端口的进程（前端）
   netstat -ano | findstr :5173
   taskkill /F /PID <进程ID>
   ```
